# daw-capa-dominio

## Descripción

Este skill se enfoca en la definición de la capa de dominio para una aplicación web utilizando Spring Boot.

### Cuando usar este skill

- Cuando se necesite definir la capa de dominio de una aplicación web.
- Cuando se quiera estructurar la lógica de negocio de una aplicación utilizando Spring Boot.

## Contexto

La capa de dominio es la parte de una aplicación web que se encarga de representar los conceptos y las reglas de negocio del dominio. En Spring Boot, la capa de dominio se puede implementar utilizando modelos de dominio, servicios, casos de uso, etc.

## Información adicional

- Usar las skills de casos de uso, servicios, modelos, validación y repositorios para definir los casos de uso, servicios, modelos, validaciones y repositorios de la capa de dominio.
- La capa de dominio debe ser independiente de la capa web y de la capa de acceso a datos, para mantener una separación de responsabilidades adecuada.
- La capa de dominio debe contener la lógica de negocio de la aplicación, mientras que la capa web debe contener la lógica de presentación y la capa de acceso a datos debe contener la lógica de persistencia.
